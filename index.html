01 — EXECUTIVE SUMMARY

What Is Edge Olfaction?

A localised, zero-latency Electronic Nose that predicts agricultural spoilage in transit — without any cloud dependency. It fuses an Arduino-driven multivariable gas sensor array with a Raspberry Pi edge inference engine.
  • Decentralised Architecture — Separates raw ADC polling (Arduino) from mathematical inference (Raspberry Pi)
  • Dynamic Cargo Profiling — Swappable algorithmic profiles per produce type (ethylene for apples, sulfur/methane for onions)
  • Transit Acoustic Filtering — Piezo impact disc array tracks baselines to distinguish truck rumble from pest activity
  • Headless Survival Mode — Immutable SSH file locks & hardcoded nmcli WPA-PSK overrides survive mobile hotspot drops
📡 Deep Dive: Headless Field Survival

Note for the Panel: Standard dynamic networking fails during mobile transit[cite: 133]. To ensure the Raspberry Pi does not drop the mobile hotspot connection, we enforce aggressive network overrides.


We hardcode the WPA-PSK via terminal nmcli commands and apply an immutable file lock to the SSH configuration using sudo chattr +i /etc/ssh/sshd_config[cite: 134, 142]. This guarantees unrestricted IPv4 SSH listening on Port 22, allowing the node to be accessible to engineers even without a GUI[cite: 139, 194, 230].

02 — PROBLEM / SOLUTION

The Dead-Zone Problem

PROBLEM
Agricultural supply chains lose millions to transit spoilage. Current IoT sensors require stable cloud APIs. If the truck loses Wi-Fi, data is buffered — and by the time the system reconnects, the entire batch of produce is already rotting.
SOLUTION
Intelligence moved to the edge. The system monitors Volatile Organic Compounds (VOCs) locally, interpolates ADC readings against calibrated Fresh / Rot constants, and executes asynchronous Telegram payloads the second it hits a hotspot.

🎛️ Deep Dive: Transit Acoustic Filtering

Note for the Panel: To prevent false positives from physical movement, the system categorizes vibrations using a Piezo impact disc array[cite: 160, 195].


Our dynamic baseline tracking mathematically filters this noise: a sudden vibration spike of >100 triggers a 5-second hardware lockout to ignore transit rumble[cite: 195, 196]. Conversely, a continuous micro-spike of >10 flags potential pest activity, successfully separating truck movement from organic threats[cite: 196].

03 — MATHEMATICAL ARCHITECTURE

Decay Models

Arrhenius Rate Equation $$k = A \, e^{-E_a / RT}$$

where $k$ = spoilage rate, $A$ = pre-exponential factor, $E_a$ = activation energy, $R$ = universal gas constant, $T$ = temperature (K)

Risk Normalisation $$\text{Risk}_\text{sensor} = \frac{V_\text{current} - V_\text{fresh}}{V_\text{rot} - V_\text{fresh}} \times 100$$
Aggregate Clamped Risk & Shelf-Life Projection $$\text{Risk}_\text{total} = \max\!\left(5.0,\, \min\!\left(99.9,\, \frac{\text{Risk}_{135} + \text{Risk}_{3}}{2}\right)\right)$$ $$\text{Hours}_\text{left} = \max\!\left(1,\, \left\lfloor 168 \times \left(\frac{100 - \text{Risk}_\text{total}}{100}\right)^{\!3}\right\rfloor\right)$$
🔬 Deep Dive: Pre-Exponential Gas Integration

Note for the Panel: Our cubic decay model ties directly into the Arrhenius equation: $$k = A \, e^{-E_a / RT}$$[cite: 181].


Instead of treating the pre-exponential factor ($A$) as a static constant, our system dynamically increases it based on the integrated MQ-135 and MQ-3 gas sensor outputs[cite: 137, 204]. This mathematically represents the accelerated rotting rate caused specifically by external ethylene or fungal off-gassing, resulting in a highly accurate exponential decay curve[cite: 204, 214].

⚙️ Deep Dive: Pipeline Dependencies

Note for the Panel: The analog-to-digital pipeline requires strict physical preparation. The MQ sensor array must undergo a 24-48 hour continuous heating "burn-in" phase to stabilize the internal resistance before any reliable calibration can occur.


On the software side, the Pi's stability depends on jarvis_bridge.py maintaining a background thread, while csv_logger.py writes ambient data locally (generating our khardaha_full_log files) without causing I/O blocking or crashing the AI risk routine.
